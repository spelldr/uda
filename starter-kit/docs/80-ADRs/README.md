# Architecture Decision Records (ADRs)

Document major decisions: why we chose X over Y, trade-offs, consequences.

**Format:** ADR-[number]: [Decision title]

Example: `ADR-0001: Why we chose PostgreSQL over MongoDB`

**Structure:**
- Context (why this decision was needed)
- Decision (what was chosen)
- Consequences (trade-offs and impacts)
- Alternatives considered

**Status:** Accepted, Pending, Rejected, or Superseded

**Links allowed:**
- ✅ To other ADRs
- ✅ To Release Notes
- ✅ To Concepts

**Duration:** ADRs are permanent (never auto-expire).

**Copy from:** `../../templates/Template - Concept.md` (adapt for ADRs)
